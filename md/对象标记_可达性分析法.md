

# 可达性分析法

    以GC Roots的对象作出发
    通过引用关系遍历对象图，
    能被遍历到的对象就被判定为存活（对象头中进行标记）
     
# GC Roots

    虚拟机栈和本地方法栈中的引用的对象
    方法区中的类静态属性引用的对象
    方法区中的常量引用的对象
    （都是当前必须存活的对象）
   
# 特点

    标记时间和存活数量成正比

# 优点

    有效的解决循环引用问题

# 缺点

    GC占用运行时时间（必须先暂停用户程序）
    

# 运用

    jvm中采用的是”可达性分析”。
