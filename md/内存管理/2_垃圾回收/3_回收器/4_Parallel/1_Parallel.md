   
# Parallel Scavenge

# 定位 
    
    新生代收集器
    吞吐量优先收集器
    （吞吐量=运行用户代码时间/CPU总时间)
    
               
# 特点

    使用复制算法
    并行收集器
    

# 优点

    停顿时间可控
    自适应调节分代区域
    
# 缺点

    对cpu资源敏感
   
        
# 适用情况
    
    
    而高吞吐量则可以最高效率地利用CPU时间，尽快地完成程序的运算任务，
    主要适合在后台运算而不需要太多交互的任务。
