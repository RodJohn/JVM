   
# 复制算法

    Copying（复制）算法

# 原理

    将可用内存按容量划分两块，每次只使用其中的一块。
    当其中一块的内存用完了，
    就用可达性分析法存活着的对象并复制到另外一块上面，
    然后再把已使用的内存空间一次清理掉，


# 图解

![](https://github.com/RodJohn/JVM/blob/master/img/gccopy.jpg)

# 优点
    
      没有内存碎片
         
      停顿时间短
         只需分析并复制
         花费的时间和存活对象成正比
    
# 缺点

      1.能使用的内存缩减到原来的一半    
      2.移动对象将STW
         （不兼容保守式GC）  
      3.当大量对象存活时，需要大量的内存复制

    
    
# 运用

    适用于对象存活率低的情况
